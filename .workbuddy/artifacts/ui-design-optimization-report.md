# AxAgent UI 设计审查与优化方案

> **审查日期**: 2026-04-30  
> **审查范围**: 全量 UI 组件与页面  
> **设计原则**: 美观、易用、符合用户习惯、一致性优先

---

## 一、当前架构概览

| 层级 | 技术栈 | 状态 |
|------|--------|------|
| 框架 | React 19 + TypeScript + Vite 8 | ✅ 现代 |
| UI 库 | Ant Design v6 + @ant-design/x v2.5 | ✅ 成熟 |
| 样式 | TailwindCSS v4 + antd-style v4 | ✅ 灵活 |
| 状态管理 | Zustand v5 | ✅ 轻量 |
| 图标 | Lucide React + @lobehub/icons | ✅ 丰富 |
| 桌面壳 | Tauri v2 | ✅ 原生 |

**组件规模**: ~70+ chat 组件、30+ settings 组件、workflow 编辑器、Wiki 系统、QuickBar 迷你模式

## 二、发现的问题清单

### 🔴 高优先级 (影响核心体验)

#### 2.1 侧边导航栏 - 纯图标、无可读性

**现状**: 左侧 48px 极窄侧边栏，仅展示图标，无文字标签。用户必须悬停等待 Tooltip 才知道每个图标的功能。

```tsx
// Sidebar.tsx - 圆形按钮，无标签
<button style={{ width: 36, height: 36, borderRadius: "50%" }}>
  <MessageSquare size={18} />
</button>
```

**问题**:
- 新用户需要逐个 hover 才能找到目标页面
- 9 个导航项用圆形图标排列，视觉区分度低
- 图标颜色硬编码（`NAV_ICON_COLORS`），暗色模式下对比度可能不足
- 没有活跃状态指示器（仅靠背景色区分）

**优化建议**:
1. 支持侧边栏展开/折叠，展开时显示图标+标签
2. 添加左侧活跃指示条（类似 VS Code Activity Bar）
3. 图标颜色迁移到语义 Token，自动适配明暗主题
4. 添加未读/通知角标支持
5. 支持拖拽重排导航项顺序

**设计方案**:
```
┌──────┬──────────────────────────┐
│ ●  💬│ 💬 对话    Chat          │ ← 展开态（200px）
│     │ ⚡ 技能    Skills         │
│  ⚡  │ 🏪 市场    Marketplace    │
│     │ ...                      │
│  🏪  │                          │
│     │                      👤   │
├──────┴──────────────────────────┤
│  48px → 200px (hover 展开)      │
└─────────────────────────────────┘
```

---

#### 2.2 TitleBar - 按钮密集、视觉噪音

**现状**: 36px 标题栏塞入了 10+ 个按钮（置顶、主题、语言、备份、GitHub、更新、刷新、设置 + Windows 窗口控制）。

**问题**:
- 所有按钮等大同质，无优先级区分
- 备份按钮显示倒计时文本，布局不稳定
- 移动端/小窗口下按钮溢出
- 主题/语言切换使用独立 Dropdown，认知负荷高

**优化建议**:
1. 将低频操作（备份详情、GitHub、检查更新）收入"更多"菜单（`⋯`）
2. 合并主题和语言为单一"偏好设置"下拉
3. 备份倒计时改为 Tooltip 显示，按钮保持固定尺寸
4. 添加分隔线区分功能组
5. 设置按钮在设置页显示关闭图标 `✕`，当前实现合理保留

**设计方案**:
```
┌─ AxAgent ─── 📌 🌓 🌐 ☁️ ⋯ 🔄 ── ⚙ ─ [─ □ ✕] ─┐
│   Logo      置顶 主题 语言 备份 更多 刷新 设置  窗口  │
└────────────────────────────────────────────────────┘
```

---

#### 2.3 对话侧边栏 - 信息密度过高

**现状**: ChatSidebar 256px 宽，展示对话列表 + 分类管理 + 拖拽排序 + 右键菜单。约 1600+ 行代码。

**问题**:
- 搜索、新建、分类筛选挤在同一行
- 对话列表项信息单一（仅标题+时间），无预览
- 拖拽分类管理学习成本高
- 无对话预览/摘要

**优化建议**:
1. 支持调整侧边栏宽度（拖拽分隔线）
2. 对话列表项增加最后一条消息预览
3. 搜索栏独立成行，增加筛选条件
4. 分类管理简化为下拉选择器（替代拖拽）
5. 添加"收藏/置顶"对话功能

---

#### 2.4 聊天输入区 - 功能堆砌

**现状**: InputArea.tsx 约 87KB，2177 行，是项目中最大的单文件组件。

**问题**:
- 输入框上方的附件/上下文栏占用空间
- 模型选择器、工作流选择器、技能选择器散落在不同位置
- 发送按钮与附加功能按钮混排
- 没有输入框自动增高限制

**优化建议**:
1. 拆分 InputArea 为多个子组件（Toolbar、AttachmentBar、ContextBar、SendButton）
2. 高级功能（工作流、技能）收入"+"菜单
3. 模型选择器移至输入框左侧内联显示
4. 输入框设置最大高度 200px，超出滚动

---

#### 2.5 全局文本选择禁用

**现状**: `index.css` 中全局设置 `user-select: none`，仅对特定元素恢复。

```css
body {
  user-select: none;
  -webkit-user-select: none;
}
```

**问题**:
- 这是桌面应用，不是移动端，不需要全局禁用选择
- 用户无法在设置页、wiki 页、文件页等复制文本
- 虽然有白名单恢复部分元素，但遗漏场景多

**优化建议**:
1. **反转策略**: 默认允许选择，仅对拖拽区域（TitleBar）、交互元素（按钮、标签）禁用
2. 对所有内容展示区域（设置表单、文档、文件列表）确保可选择

---

### 🟡 中优先级 (影响使用效率)

#### 2.6 TabBar - 空间利用不足

**现状**: 34px 高度，标签最大宽度 200px，标题截断为 24 字符。右侧有大片空白区域。

**问题**:
- 标签高度偏低，关闭按钮点击区域小
- 无法看到完整标题
- 多余的空白区域未利用（可放置全局操作）

**优化建议**:
1. 增加标签高度至 36px，增大操作区域
2. 支持标签宽度自适应（flexible width）
3. 标签 hover 时显示完整标题 Tooltip
4. 右侧空白区域可放置"分支对比"、"历史"等全局操作

---

#### 2.7 设置页面 - 导航可发现性差

**现状**: SettingsSidebar 包含 19 个设置项，无搜索功能。用户需要逐个扫描。

**问题**:
- 设置项数量多，无分组/分类
- 无搜索过滤功能
- 部分设置项命名不一致（中英混合）

**优化建议**:
1. 顶部添加搜索框，实时过滤设置项
2. 将 19 项分为 4-5 个功能组（通用、模型、数据、工具、系统）
3. 添加最近访问记录
4. 设置项名称统一风格

---

#### 2.8 空状态与加载态

**现状**: 多个页面缺少空状态设计。ContentArea 使用通用 `<Spin>` 加载。

**问题**:
- 新用户首次打开知识库/记忆/文件页面看到空白
- 无引导性空状态（CTA 按钮）
- 加载态统一使用 Ant Design Spin，无骨架屏

**优化建议**:
1. 为每个功能页面设计空状态插画 + 引导文案 + CTA 按钮
2. 对话列表加载使用骨架屏（Skeleton）
3. 聊天消息流式加载使用逐字显示动画

---

#### 2.9 面包屑导航被隐藏

**现状**: BreadcrumbBar 仅在子会话（SubAgent）场景显示，父会话不可见。

**问题**:
- 普通用户看不到对话层级关系
- 子会话之间导航不直观

**优化建议**:
1. 面包屑始终显示（主对话显示"🏠 对话名"）
2. 子会话自动追加层级
3. 支持点击面包屑任意层级跳转

---

### 🟢 低优先级 (提升品质)

#### 2.10 动画与过渡

**现状**: 项目仅使用基础的 hover 颜色过渡和少量 CSS 动画。

**可优化点**:
- 页面切换无过渡动画
- 消息出现无动画
- 面板展开/折叠无动画
- 按钮按下无反馈动画

**优化建议**:
1. 页面切换添加淡入淡出（fade + slide）
2. 新消息添加从下往上的滑入动画
3. 面板使用 `transform: scaleY` 展开
4. 按钮添加 `:active` 缩放反馈

---

#### 2.11 色彩系统一致性

**现状**: 图标颜色通过 `NAV_ICON_COLORS`、`TITLEBAR_ICON_COLORS`、`SETTINGS_ICON_COLORS` 三套独立常量管理。

**问题**:
- 暗色模式下部分图标颜色可能不可见
- 三套颜色系统维护成本高
- 非 Ant Design Token，无法自动适配主题

**优化建议**:
1. 统一为单一 `ICON_COLORS` 常量文件
2. 所有颜色值使用 Ant Design Token 或 CSS 变量
3. 确保 WCAG AA 对比度标准（明暗模式均 ≥ 4.5:1）

---

#### 2.12 快捷键可视化

**现状**: 部分按钮的 Tooltip 中显示了快捷键（如 `Ctrl+K`），但不是全部。

**优化建议**:
1. 全局添加 `Ctrl+/` 或 `?` 快捷键面板
2. 菜单项统一显示快捷键
3. 命令面板（CommandPalette）支持更多操作

---

## 三、设计系统建议

### 3.1 色彩方案

```
┌─────────────────────────────────────────────────┐
│  语义色板 (Semantic Palette)                      │
│                                                  │
│  主色    #1677ff  主要操作、链接、活跃状态          │
│  成功    #52c41a  完成状态、正向反馈               │
│  警告    #faad14  需要注意、进行中                 │
│  危险    #ff4d4f  错误、删除、破坏性操作            │
│  信息    #1890ff  中性信息提示                     │
│                                                  │
│  中性色板 (Neutral)                               │
│  bg-container  #ffffff / #141414                 │
│  bg-elevated   #ffffff / #1f1f1f                 │
│  text          rgba(0,0,0,0.88) / rgba(255,..)  │
│  text-secondary rgba(0,0,0,0.45)                 │
│  border        #f0f0f0 / #303030                 │
└─────────────────────────────────────────────────┘
```

### 3.2 间距体系

沿用 Ant Design 4px 基准网格：`4, 8, 12, 16, 20, 24, 32, 48`

### 3.3 圆角体系

基于用户可配置的 `borderRadius`（默认 8px），派生：
- `radiusXS`: 2px（标签、徽章）
- `radiusSM`: 5px（小按钮、输入框）
- `radius` : 8px（卡片、面板）
- `radiusLG`: 11px（模态框）

### 3.4 动效规范

| 类型 | 时长 | 缓动 |
|------|------|------|
| hover 过渡 | 150ms | ease-in-out |
| 页面切换 | 200ms | ease-out |
| 面板展开 | 250ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 消息出现 | 300ms | ease-out |
| 模态框 | 200ms | cubic-bezier(0.4, 0, 0.2, 1) |
