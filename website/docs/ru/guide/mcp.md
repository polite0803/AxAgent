# MCP-серверы

## Что такое MCP?

[Model Context Protocol (MCP)](https://modelcontextprotocol.io/) — открытый стандарт, позволяющий ИИ-моделям взаимодействовать с внешними инструментами и источниками данных. AxAgent выступает как MCP-клиент — вы добавляете MCP-серверы, и ИИ может вызывать предоставляемые ими инструменты во время разговора.

---

## Транспортные протоколы

AxAgent поддерживает три транспортных протокола для связи с MCP-серверами:

| Протокол | Соединение | Сценарий использования | Конфигурация |
|---------|-----------|----------------------|-------------|
| **Stdio** | Локальный процесс | Инструменты на вашей машине, запускаемые через `npx`, `uvx`, `python` и т.д. | `command` + `args` + опциональный `env` |
| **SSE** | Удалённый сервер | Server-Sent Events endpoint на удалённой машине | `url` |
| **StreamableHTTP** | Удалённый сервер | HTTP-потоковый endpoint, более новая альтернатива SSE | `url` |

## Добавление MCP-серверов

### Создание через форму

1. Перейдите в **Настройки → MCP-серверы**.
2. Нажмите **Добавить MCP-сервер**.
3. Введите имя и выберите транспортный протокол.
4. Заполните поля для выбранного протокола.
5. Нажмите **Сохранить**.

### Импорт JSON

Нажмите **Импорт JSON** и вставьте объект конфигурации:

```json
{
  "mcpServers": {
    "fetch": {
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    }
  }
}
```

## Встроенные инструменты

| Инструмент | Описание |
|-----------|---------|
| **@axagent/fetch** | Загрузка веб-страниц и HTTP-ресурсов |
| **@axagent/search-file** | Поиск файлов в локальной файловой системе |
