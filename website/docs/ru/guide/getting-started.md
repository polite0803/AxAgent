# Быстрый старт

## Установка

Скачайте последний установщик со [страницы загрузки](/ru/download) или [GitHub Releases](https://github.com/polite0803/AxAgent/releases).

### macOS

| Чип | Файл |
|-----|------|
| Apple Silicon (M1 / M2 / M3 / M4) | `AxAgent_x.x.x_aarch64.dmg` |
| Intel | `AxAgent_x.x.x_x64.dmg` |

1. Откройте `.dmg` и перетащите **AxAgent** в папку **«Программы»**.
2. Запустите AxAgent. Если macOS блокирует приложение, перейдите в **Системные настройки → Конфиденциальность и безопасность** и нажмите **Всё равно открыть**.

::: warning macOS: «Приложение повреждено» или «Разработчик не верифицирован»
Если вы видите одно из этих сообщений, откройте Терминал и выполните:

```bash
xattr -c /Applications/AxAgent.app
```

Затем снова запустите приложение. Это удаляет флаг карантина, который macOS применяет к неподписанным загрузкам.
:::

### Windows

| Архитектура | Файл |
|------------|------|
| x64 (большинство ПК) | `AxAgent_x.x.x_x64-setup.exe` |
| ARM64 | `AxAgent_x.x.x_arm64-setup.exe` |

Запустите установщик и следуйте мастеру. Запустите AxAgent из меню «Пуск» или ярлыка на рабочем столе.

### Linux

| Формат | Архитектура | Файл |
|--------|------------|------|
| Debian / Ubuntu | x64 | `AxAgent_x.x.x_amd64.deb` |
| Debian / Ubuntu | ARM64 | `AxAgent_x.x.x_arm64.deb` |
| Fedora / openSUSE | x64 | `AxAgent_x.x.x_x86_64.rpm` |
| Fedora / openSUSE | ARM64 | `AxAgent_x.x.x_aarch64.rpm` |
| Любой дистрибутив | x64 | `AxAgent_x.x.x_amd64.AppImage` |
| Любой дистрибутив | ARM64 | `AxAgent_x.x.x_aarch64.AppImage` |

```bash
# Debian / Ubuntu
sudo dpkg -i AxAgent_x.x.x_amd64.deb

# Fedora / openSUSE
sudo rpm -i AxAgent_x.x.x_x86_64.rpm

# AppImage (любой дистрибутив)
chmod +x AxAgent_x.x.x_amd64.AppImage
./AxAgent_x.x.x_amd64.AppImage
```

## Начальная настройка

### 1. Открыть настройки

Запустите AxAgent и нажмите на **значок шестерёнки** внизу боковой панели, или нажмите <kbd>Cmd/Ctrl</kbd>+<kbd>,</kbd>.

### 2. Добавить провайдера

Перейдите в **Настройки → Провайдеры** и нажмите кнопку **+**.

1. Введите отображаемое имя (например, *OpenAI*).
2. Выберите тип провайдера (OpenAI, Anthropic, Google Gemini и т.д.).
3. Вставьте ваш API-ключ.
4. Подтвердите **Base URL** — официальный endpoint предзаполнен для встроенных типов.

::: tip
Вы можете добавить столько провайдеров, сколько захотите. Каждый провайдер управляет своими ключами API и моделями независимо.
:::

### 3. Получить модели

Нажмите **Получить модели**, чтобы загрузить список доступных моделей из API провайдера. Вы также можете добавлять ID моделей вручную при необходимости.

### 4. Установить модель по умолчанию

Перейдите в **Настройки → Модель по умолчанию** и выберите провайдера и модель, которые новые разговоры должны использовать по умолчанию.

---

## Ваш первый разговор

1. Нажмите **Новый чат** в боковой панели (или нажмите <kbd>Cmd/Ctrl</kbd>+<kbd>N</kbd>).
2. Выберите модель из селектора моделей вверху чата.
3. Введите сообщение и нажмите <kbd>Enter</kbd>.
4. AxAgent транслирует ответ в реальном времени. Модели с поддержкой блоков размышлений (например, Claude, DeepSeek R1) отображают процесс рассуждений в сворачиваемом разделе.

## Горячие клавиши

| Сочетание | Действие |
|-----------|---------|
| <kbd>Cmd/Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>A</kbd> | Показать / скрыть текущее окно |
| <kbd>Cmd/Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>Alt</kbd>+<kbd>A</kbd> | Показать / скрыть все окна |
| <kbd>Cmd/Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>W</kbd> | Закрыть окно |
| <kbd>Cmd/Ctrl</kbd>+<kbd>N</kbd> | Новый разговор |
| <kbd>Cmd/Ctrl</kbd>+<kbd>,</kbd> | Открыть настройки |
| <kbd>Cmd/Ctrl</kbd>+<kbd>K</kbd> | Палитра команд |
| <kbd>Cmd/Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>G</kbd> | Переключить API-шлюз |
